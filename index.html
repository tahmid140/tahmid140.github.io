Late materialization for multi way stream joins

Kotlin Apache Storm Postgres

My master thesis was based on optimizing join performance for distributed stream join computations. The idea of the thesis was to apply different materialization techniques and prove that in certain cases late materialization of join results can be beneficial to performance. Using Apache Storm and Kotlin I implemented topologies with both late and early result materialization. It was observed that, with low parallelism in the storm cluster, late materialization performed better. I worked on my thesis at the DBIS lab in TU Kaiserslautern. Take a look at my thesis report to learn more!

Web Search Engine

Java SE Java EE Postgres

For my master project I developed a web search engine from scratch. This was a very interesting project for me because it gave me a strong insight on how real life search engines actually operate. The project consisted of a multi-threaded crawler which would crawl web pages starting from a seed URL in a BFS fashion. The content of each page was tokenized, filtered and stored in a database. Document relevance for search queries was determined using TF-IDF and PageRank. Database level indices were also used to improve query performance. Finally a meta-search interface was also developed which combined search results from the databases of other groups as well.
